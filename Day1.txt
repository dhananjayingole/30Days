## **PART 1: UNDERSTANDING ACTIVITY LIFECYCLE (45 mins)**

### **What is an Activity?**

An **Activity** is a single screen with a user interface in an Android app. It's one of the core components of Android. Think of it like a screen in your app:
- Login screen = Activity
- Home screen = Activity  
- Settings screen = Activity
- Chat screen = Activity (like in your Chatbot app)

### **Why Do We Need Lifecycle?**

Activities go through different states. When you open an app, the activity is created. When you close it, it's destroyed. Between these, many things happen. Understanding this helps you:
- Save user data before closing
- Release resources (camera, GPS, database connections)
- Resume work when user comes back
- Handle configuration changes (like rotating phone)

---

## **📊 THE ACTIVITY LIFECYCLE STATES & CALLBACKS**

Here's the journey of an Activity (in order):

```
┌─────────────────────────────────────────────────────────────────┐
│                      ACTIVITY LIFECYCLE                          │
└─────────────────────────────────────────────────────────────────┘

                    (App is Launched)
                          ↓
                    ┌──────────────┐
                    │  onCreate()  │  ← Activity is created for FIRST time
                    └──────┬───────┘    - Initialize variables
                           ↓            - Set up UI (setContentView)
                    ┌──────────────┐
                    │ onStart()    │  ← Activity becomes VISIBLE
                    └──────┬───────┘    - User can see the screen
                           ↓            - But NOT yet interacting
                    ┌──────────────┐
                    │ onResume()   │  ← Activity is in FOREGROUND (ACTIVE)
                    └──────┬───────┘    - User can interact
                           ↓            - Start animations, music, GPS
                    
                  *** USER IS USING APP ***
                  
                    ┌──────────────┐
                    │ onPause()    │  ← Activity losing focus (another app comes)
                    └──────┬───────┘    - PAUSE music, animations
                           ↓            - Save temporary data
                    ┌──────────────┐
                    │ onStop()     │  ← Activity is NO LONGER VISIBLE
                    └──────┬───────┘    - User switched to another app
                           ↓            - Release expensive resources
                    ┌──────────────┐
                    │ onDestroy()  │  ← Activity is DESTROYED
                    └──────────────┘    - Final cleanup before destruction
                           
                    *** ACTIVITY OBJECT IS REMOVED ***
```

---

## **🔄 DETAILED BREAKDOWN OF EACH CALLBACK**

### **1️⃣ onCreate() - Birth of Activity**

**Called:** Only ONCE when activity is created for the first time (not called on rotation by default)

**What to do here:**
- Initialize UI components (buttons, text fields)
- Set up the layout using `setContentView()`
- Initialize variables
- Restore saved state (if activity was killed before)

**Example:**
```kotlin
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    setContentView(R.layout.activity_main)  // Set the layout file
    
    // Initialize views
    val button: Button = findViewById(R.id.my_button)
    button.setOnClickListener { 
        Toast.makeText(this, "Button Clicked!", Toast.LENGTH_SHORT).show()
    }
    
    // Initialize variables
    var counter = 0
}
```

**Real example from your projects:**
In your **Chatbot app**, when the chat screen opens:
- `onCreate()` loads the chat UI
- Sets up message adapter
- Connects to the RAG backend

---

### **2️⃣ onStart() - Activity Becomes Visible**

**Called:** After `onCreate()`, AND whenever activity comes back to foreground

**What happens:**
- Activity UI is now visible to user
- But user CANNOT interact yet (UI might not be fully ready)

**What to do here:**
- Start animations that show the screen
- Load non-critical data for display
- Register broadcast receivers
- Start location updates (but pause in onPause)

**Example:**
```kotlin
override fun onStart() {
    super.onStart()
    Log.d("MyActivity", "Activity is now visible")
    // Start listening for location updates
    startLocationUpdates()
}
```

---

### **3️⃣ onResume() - Activity is Active & Interactive**

**Called:** After `onStart()`, AND when coming back from `onPause()`

**What happens:**
- Activity is now in FOREGROUND
- User CAN interact with the screen
- This is the active, running state

**What to do here:**
- Start animations
- Start playing music/video
- Request camera access
- Update UI with real-time data
- Resume sensors (accelerometer, GPS)

**Example:**
```kotlin
override fun onResume() {
    super.onResume()
    Log.d("MyActivity", "Activity is active and user can interact")
    
    // Start camera preview (if this is a camera app)
    startCameraPreview()
    
    // Start playing music
    mediaPlayer.start()
    
    // Update UI with latest data
    loadUserData()
}
```

**Real example from your projects:**
In your **Patient Management App**, when the patient list screen resumes:
- Fetch latest patient data
- Refresh the RecyclerView
- Restore scroll position

---

### **4️⃣ onPause() - Activity Losing Focus**

**Called:** When another activity comes on top, or app goes to background

**What happens:**
- Activity is STILL VISIBLE (if another activity is transparent)
- But user is NOT interacting with it
- Another activity is taking focus

**What to do here:**
- PAUSE music/video playback
- PAUSE animations (conserve battery)
- Save temporary data (user input in text fields)
- Release heavy resources (camera, sensors)
- Do NOT do heavy operations here (must be QUICK)

**Example:**
```kotlin
override fun onPause() {
    super.onPause()
    Log.d("MyActivity", "Activity is losing focus")
    
    // Pause music
    mediaPlayer.pause()
    
    // Stop camera
    stopCameraPreview()
    
    // Save user input
    val userInput = editText.text.toString()
    saveToPreferences(userInput)
}
```

**Important:** `onPause()` should complete in milliseconds. Don't do database writes here - do that in `onStop()`.

---

### **5️⃣ onStop() - Activity is Not Visible**

**Called:** When activity is completely hidden (another app is now fullscreen)

**What happens:**
- Activity is NO LONGER VISIBLE
- App might be in background or minimized
- This is the time to release expensive resources

**What to do here:**
- Save data to database/SharedPreferences
- Stop location updates
- Close database connections
- Stop services that aren't needed when not visible
- Unregister receivers

**Example:**
```kotlin
override fun onStop() {
    super.onStop()
    Log.d("MyActivity", "Activity is no longer visible")
    
    // Save important data to database
    saveChatMessagesToDB()
    
    // Stop location tracking
    stopLocationUpdates()
    
    // Close database connection
    database.close()
}
```

---

### **6️⃣ onDestroy() - Activity is Being Destroyed**

**Called:** When activity is being destroyed (user pressed back, or system killed it)

**What happens:**
- Activity is about to be removed from memory
- This might be the last chance to clean up

**What to do here:**
- Final cleanup
- Release resources not released in onStop()
- This is NOT guaranteed to be called (system might kill process)

**Example:**
```kotlin
override fun onDestroy() {
    super.onDestroy()
    Log.d("MyActivity", "Activity is being destroyed")
    
    // Cancel pending operations
    handler.removeCallbacks(runnable)
    
    // Release thread resources
    cleanUpThreads()
}
```

---

## **🔄 IMPORTANT SCENARIOS**

### **Scenario 1: User Opens Your App**
```
onCreate() → onStart() → onResume()
[User interacts with app]
```

### **Scenario 2: User Receives Phone Call**
```
[In onResume()] → onPause() → onStop()
[After call ends] → onStart() → onResume()
```

### **Scenario 3: User Presses Back Button**
```
[In onResume()] → onPause() → onStop() → onDestroy()
[Activity is removed, new activity shows]
```

### **Scenario 4: User Rotates Phone (Screen Rotation)**
```
[In onResume()] → onPause() → onStop() → onDestroy() 
→ onCreate() → onStart() → onResume()
[A NEW activity instance is created with new orientation]
```

### **Scenario 5: System Kills Activity (Low Memory)**
```
[In onStop()] → onDestroy() [NO onPause/onStop called!]
[User opens app again] → onCreate() → onStart() → onResume()
[Your saved data (if any) is restored in onCreate()]
```

---

## **📌 KEY POINTS TO REMEMBER**

| Callback | Visible? | Interactive? | Frequency | Use For |
|----------|----------|--------------|-----------|---------|
| `onCreate()` | ❌ | ❌ | Once per launch | Initialize UI, variables |
| `onStart()` | ✅ | ❌ | Multiple | Show UI, register receivers |
| `onResume()` | ✅ | ✅ | Multiple | Start animations, sensors |
| `onPause()` | ✅ | ❌ | Multiple | Pause music, save temp data |
| `onStop()` | ❌ | ❌ | Multiple | Save data, close DB |
| `onDestroy()` | ❌ | ❌ | Once | Final cleanup |

---

## **⚠️ COMMON MISTAKES**

❌ **Mistake 1:** Doing heavy database operations in `onPause()`
✅ **Fix:** Do in `onStop()` instead

❌ **Mistake 2:** Not saving data in `onStop()`, expecting `onDestroy()` to be called
✅ **Fix:** System might kill process without calling `onDestroy()`

❌ **Mistake 3:** Not pausing expensive operations (music, GPS) in `onPause()`
✅ **Fix:** Always pause in `onPause()`, resume in `onResume()`

# **PART 2: INTENT SYSTEM (30 mins)**

## **What is an Intent?**

An **Intent** is a message object that you use to request an action from another app component. It's like:
- "Hey Android, open a website"
- "Hey Android, send a message"
- "Hey Android, open the camera"
- "Hey Android, navigate to another screen"

### **2 Types of Intents:**

### **Type 1: EXPLICIT INTENT** (Know exactly which app/component to open)

```kotlin
// Open LoginActivity from MainActivity
val intent = Intent(this, LoginActivity::class.java)
startActivity(intent)
```

**Structure:**
```
Intent(context, targetActivityClass)
```

**With Data:**
```kotlin
val intent = Intent(this, ChatActivity::class.java)
intent.putExtra("user_id", 123)
intent.putExtra("user_name", "John")
startActivity(intent)
```

**Receiving Data in Target Activity:**
```kotlin
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    setContentView(R.layout.activity_chat)
    
    // Get the data
    val userId = intent.getIntExtra("user_id", -1)
    val userName = intent.getStringExtra("user_name") ?: ""
    
    Log.d("ChatActivity", "Received: $userId, $userName")
}
```

---

### **Type 2: IMPLICIT INTENT** (Let Android choose which app to open)

```kotlin
// Open a website - Android will choose which browser to use
val intent = Intent(Intent.ACTION_VIEW)
intent.data = Uri.parse("https://www.google.com")
startActivity(intent)
```

**Common Implicit Intents:**

```kotlin
// Open Phone Dialer
val intent = Intent(Intent.ACTION_DIAL)
intent.data = Uri.parse("tel:9876543210")
startActivity(intent)

// Send Email
val intent = Intent(Intent.ACTION_SEND)
intent.type = "message/rfc822"
intent.putExtra(Intent.EXTRA_EMAIL, arrayOf("test@gmail.com"))
intent.putExtra(Intent.EXTRA_SUBJECT, "Hello")
intent.putExtra(Intent.EXTRA_TEXT, "This is a test email")
startActivity(intent)

// Open Camera
val intent = Intent(MediaStore.ACTION_IMAGE_CAPTURE)
startActivityForResult(intent, CAMERA_REQUEST_CODE)

// Open File Picker
val intent = Intent(Intent.ACTION_PICK)
intent.type = "image/*"
startActivityForResult(intent, PICK_IMAGE_REQUEST_CODE)
```

---

## **Intent Filters: Making Your App Respond to Intents**

In your `AndroidManifest.xml`, you can declare that your activity can handle certain actions:

```xml
<activity android:name=".ChatActivity">
    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <category android:name="android.intent.category.BROWSABLE" />
        <data android:scheme="https" android:host="mychatapp.com" />
    </intent-filter>
</activity>
```

This means: "If someone clicks a link like `https://mychatapp.com`, open my ChatActivity"

# **PART 3: HANDS-ON PRACTICE (45 mins)**

## **Project: Build a Simple Navigation App**

### **Step 1: Create Two Activities**

**Activity 1: MainActivity.kt**
```kotlin
package com.example.lifecycledemo

import android.content.Intent
import android.os.Bundle
import android.widget.Button
import android.widget.EditText
import android.util.Log
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        Log.d("MainActivity", "onCreate() called")
        
        val userNameInput = findViewById<EditText>(R.id.et_username)
        val sendButton = findViewById<Button>(R.id.btn_send)
        
        sendButton.setOnClickListener {
            val userName = userNameInput.text.toString()
            
            // Create explicit intent to go to SecondActivity
            val intent = Intent(this, SecondActivity::class.java)
            intent.putExtra("user_name", userName)
            intent.putExtra("user_id", 42)
            startActivity(intent)
        }
    }
    
    override fun onStart() {
        super.onStart()
        Log.d("MainActivity", "onStart() called")
    }
    
    override fun onResume() {
        super.onResume()
        Log.d("MainActivity", "onResume() called - Activity is ACTIVE")
    }
    
    override fun onPause() {
        super.onPause()
        Log.d("MainActivity", "onPause() called - Activity is PAUSING")
    }
    
    override fun onStop() {
        super.onStop()
        Log.d("MainActivity", "onStop() called - Activity is STOPPING")
    }
    
    override fun onDestroy() {
        super.onDestroy()
        Log.d("MainActivity", "onDestroy() called - Activity is DESTROYED")
    }
}
```

**Activity 2: SecondActivity.kt**
```kotlin
package com.example.lifecycledemo

import android.os.Bundle
import android.widget.Button
import android.widget.TextView
import android.util.Log
import androidx.appcompat.app.AppCompatActivity

class SecondActivity : AppCompatActivity() {
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_second)
        Log.d("SecondActivity", "onCreate() called")
        
        // Receive data from MainActivity
        val userName = intent.getStringExtra("user_name") ?: "Unknown"
        val userId = intent.getIntExtra("user_id", -1)
        
        val welcomeText = findViewById<TextView>(R.id.tv_welcome)
        welcomeText.text = "Welcome, $userName (ID: $userId)"
        
        val backButton = findViewById<Button>(R.id.btn_back)
        backButton.setOnClickListener {
            finish()  // Close this activity and go back to MainActivity
        }
    }
    
    override fun onResume() {
        super.onResume()
        Log.d("SecondActivity", "onResume() called - Activity is ACTIVE")
    }
    
    override fun onPause() {
        super.onPause()
        Log.d("SecondActivity", "onPause() called")
    }
    
    override fun onStop() {
        super.onStop()
        Log.d("SecondActivity", "onStop() called")
    }
    
    override fun onDestroy() {
        super.onDestroy()
        Log.d("SecondActivity", "onDestroy() called")
    }
}
```
