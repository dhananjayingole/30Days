## **PART 1: WHAT ARE FRAGMENTS? (15 mins)**

### **Why Do We Need Fragments?**

Imagine your **Patient Management System** (your actual project):
- You have an Admin dashboard with multiple tabs: Patients, Appointments, Prescriptions, Emergency Alerts
- Each tab is basically a different screen
- But they're all part of ONE activity

**Without Fragments:** You'd need 4 separate Activities - wasteful and complex
**With Fragments:** One Activity + 4 lightweight Fragments = Much better!

### **What is a Fragment?**

A **Fragment** is a reusable piece of UI that lives inside an Activity. Key points:

```
┌──────────────────────────────────────┐
│          Activity (Container)        │
├──────────────────────────────────────┤
│  Fragment 1  │  Fragment 2           │
│  (Patient    │  (Appointments)       │
│   List)      │                       │
└──────────────────────────────────────┘
```

- **Fragment != Activity**: Can't stand alone, needs an Activity
- **Reusable**: Same fragment can be used in multiple activities
- **Has its own lifecycle**: Similar to Activity but slightly different
- **Can communicate**: Can pass data to/from other fragments
- **Lightweight**: Multiple fragments = less memory than multiple activities

---

## **FRAGMENT LIFECYCLE**

Fragments have 9 callbacks (more than activities):

```
                onAttach()
                    ↓
                onCreate()
                    ↓
                onCreateView()  ← Create UI here (return root view)
                    ↓
                onViewCreated()  ← Initialize views here (safer than onCreateView)
                    ↓
                onStart()
                    ↓
                onResume()  ← ACTIVE
                    
            *** USER USING APP ***
            
                onPause()
                    ↓
                onStop()
                    ↓
                onDestroyView()  ← Views are destroyed
                    ↓
                onDestroy()
                    ↓
                onDetach()  ← Fragment removed from Activity
```

### **Key Callbacks Explained:**

| Callback | Purpose | Create UI? |
|----------|---------|-----------|
| `onAttach()` | Fragment attached to activity | ❌ |
| `onCreate()` | Fragment created (one-time setup) | ❌ |
| `onCreateView()` | Inflate and return the UI layout | ✅ |
| `onViewCreated()` | View is created, initialize UI components | ✅ |
| `onStart()` | Fragment becoming visible | ❌ |
| `onResume()` | Fragment is active | ❌ |
| `onPause()` | Fragment losing focus | ❌ |
| `onStop()` | Fragment no longer visible | ❌ |
| `onDestroyView()` | Views are being destroyed | ❌ |
| `onDestroy()` | Fragment being destroyed | ❌ |
| `onDetach()` | Fragment removed from activity | ❌ |

---

## **IMPORTANT DIFFERENCE: Activity vs Fragment**

| Aspect | Activity | Fragment |
|--------|----------|----------|
| Can exist alone? | ✅ Yes | ❌ No, needs Activity |
| Set layout with | `setContentView()` | `LayoutInflater.inflate()` |
| Get context | `this` | `requireContext()` or `this` |
| Start another activity | `startActivity()` | `requireActivity().startActivity()` |
| Back button | Handled automatically | Need to override in Activity |
| Memory usage | Heavy | Light |

```kotlin
import android.view.View
import android.view.ViewGroup
```

```kotlin
class PatientAdapter(

        val nameText: TextView = itemView.findViewById(R.id.tv_name)

            ageText.text = "Age: ${patient.age}"
            bloodText.text = "Blood: ${patient.bloodGroup}"
            
            itemView.setOnClickListener {
                onPatientClick(patient)
            }
        }
    }
    
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): PatientViewHolder {
        val itemView = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_patient, parent, false)
        return PatientViewHolder(itemView)
    }
    
    override fun onBindViewHolder(holder: PatientViewHolder, position: Int) {
        holder.bind(patients[position])
    }
    
    override fun getItemCount() = patients.size
}
```

## **PART 3: NAVIGATION BETWEEN FRAGMENTS (30 mins)**

Now, let's learn how to navigate from one fragment to another. There are **2 main ways**:

### **Way 1: Traditional FragmentManager (Older)**

```kotlin
// In Activity or Fragment
supportFragmentManager.beginTransaction()
    .replace(R.id.fragment_container, PatientListFragment())
    .addToBackStack(null)  // Add to back stack so back button works
    .commit()
```

### **Way 2: Jetpack Navigation (Modern - RECOMMENDED)**

Your **PatientmangSystemApp** uses this! It's much cleaner.

#### **What is Jetpack Navigation?**

A system that:
- Manages fragment transactions automatically
- Handles back navigation
- Passes data between fragments safely
- Creates navigation graphs (visual navigation flows)

#### **How to Use Jetpack Navigation:**

### **Step 1: Create Navigation Graph (nav_graph.xml)**

```xml
<?xml version="1.0" encoding="utf-8"?>
<navigation xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/nav_graph"
    app:startDestination="@id/patientListFragment">
    
    <!-- First fragment -->
    <fragment
        android:id="@+id/patientListFragment"
        android:name="com.example.patientapp.ui.fragments.PatientListFragment"
        android:label="Patients">
        
        <!-- Action to go to detail screen -->
        <action
            android:id="@+id/action_patientList_to_patientDetail"
            app:destination="@id/patientDetailFragment" />
    </fragment>
    
    <!-- Second fragment -->
    <fragment
        android:id="@+id/patientDetailFragment"
        android:name="com.example.patientapp.ui.fragments.PatientDetailFragment"
        android:label="Patient Details">
        
        <!-- Accept arguments -->
        <argument
            android:name="patientId"
            app:type="integer" />
        
        <!-- Action to go back -->
        <action
            android:id="@+id/action_patientDetail_to_patientList"
            app:destination="@id/patientListFragment" />
    </fragment>
    
</navigation>
```

## **FRAGMENT vs JETPACK COMPOSE Navigation**

### **Your PatientmangSystemApp Uses Compose Navigation:**

Since you're using **Jetpack Compose**, the navigation is different:

```kotlin
// In your composable
@Composable
fun AdminDashboardScreen(
    onNavigateTo: (String) -> Unit
) {
    Button(onClick = { 
        onNavigateTo("patient_list")  // Navigate to patient list
    }) {
        Text("View Patients")
    }
}

// Route definition
sealed class Route(val path: String) {
    object AdminDashboard : Route("admin_dashboard")
    object PatientList : Route("patient_list")
    object PatientDetail : Route("patient_detail/{patientId}") {
        fun createRoute(patientId: Int) = "patient_detail/$patientId"
    }
}

// NavHost setup
@Composable
fun NavGraph() {
    val navController = rememberNavController()
    NavHost(navController = navController, startDestination = Route.AdminDashboard.path) {
        composable(Route.AdminDashboard.path) {
            AdminDashboardScreen { destination ->
                navController.navigate(destination)
            }
        }
        composable(Route.PatientList.path) {
            PatientListScreen()
        }
    }
}
```

```kotlin
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): PatientViewHolder {
        val itemView = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_patient, parent, false)
        return PatientViewHolder(itemView)
    }
    
    override fun onBindViewHolder(holder: PatientViewHolder, position: Int) {
        holder.bind(patients[position])
    }
    
    override fun getItemCount() = patients.size
}
```

## **📝 KEY TAKEAWAYS**

| Topic | Key Point |
|-------|-----------|
| **Fragment** | Reusable UI component inside Activity |
| **Fragment Lifecycle** | `onCreateView()` for UI, `onViewCreated()` for initialization |
| **Jetpack Navigation** | Modern, safe way to navigate between fragments |
| **Safe Args** | Type-safe way to pass data between fragments |
| **NavController** | The object that handles navigation actions |
| **Back Stack** | Automatic handling with Jetpack Navigation |
